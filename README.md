# Reception App

Flutter reception/check-in app backed by Supabase.

## Supabase Setup

1. Create a Supabase project from the [Supabase dashboard](https://supabase.com/dashboard).
2. Copy `.env.example` to `.env`.
3. Set:
   - `SUPABASE_URL` from **Project Settings -> API -> Project URL**
   - `SUPABASE_ANON_KEY` from **Project Settings -> API -> anon public key**
4. Create the required tables by running the SQL below in the Supabase SQL editor.
5. Run `flutter pub get`.
6. Start the app with `flutter run`.

The app throws a clear startup error if `.env` still has placeholder values or the reception tables have not been created yet.

## Required Tables

Create these tables in Supabase SQL editor:

```sql
create table if not exists public.people (
  id text primary key,
  first_name text not null,
  last_name text not null
);

create table if not exists public.checkins (
  id bigint generated by default as identity primary key,
  person_id text not null references public.people (id) on delete cascade,
  first_name text not null,
  last_name text not null,
  checked_in_at timestamptz not null default now(),
  notes text not null default ''
);
```

If your Supabase project already has a `public.profiles` table, keep it. This app still requires the dedicated `public.people` and `public.checkins` tables above.
